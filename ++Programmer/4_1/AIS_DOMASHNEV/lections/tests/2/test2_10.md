### 1. На сервере serv01 хранится зона DNS companyA.local. Фрагмент содержимого этой зоны приведен ниже

 ┌──┬──────────────────────────────────────────────────────────┐
 │1 │@       IN  SOA serv02.companyA.local.hostmaster (        │
 │2 │        3       ; serial number                           │
 │3 │        900     ; refresh                                 │
 │4 │        600     ; retry                                   │
 │5 │        86400   ; expire                                  │
 │6 │        3600    ; default TTL                             │
 │7 │@           NS      serv01.companyE.local.                │
 │8 │@           NS      serv02.companyE.local.                │
 │9 │                                                          │
 │10│serv01      A       192.168.99.1                          │
 │11│serv02      A       192.168.99.2                          │
 │12│pc01        A       102.168.99.20                         │
 │13│)                                                         │
 └──┴──────────────────────────────────────────────────────────┘

Укажите тип этой зоны
a) Первичная
```
    b) Вторичная
```
c) Третичная
d) Зона-заглушки


___ Пояснение
Тип зоны определяется по тому, какой сервер является основным (primary) для неё, то есть где хранится оригинальная копия данных зоны.
В данном фрагменте зона DNS содержит SOA-запись, где указано, что основной сервер зоны — это serv02 (@ IN SOA serv02.companyA.local). Это означает, что сервер serv01 не является основным для этой зоны, а только хранит её копию.
Таким образом, сервер serv01 хранит вторичную копию зоны, полученную от основного сервера serv02.

### 2. DNS-сервер serv01 вышел из строя 15.05.2010 в 15:20:13. Последняя синхронизация зоны DNS companyF.local была проведена DNS-сервером serv02 15.05.2010 в 15:14:00. Фрагмент содержимого зоны companyF.local, расположенной на DNS-сервере serv02, приведен ниже.

┌──┬──────────────────────────────────────────────────┐
│1 │@       IN  SOA serv01.companyF.local.hostmaster (│
│2 │             2010    ; serial number              │
│3 │             900     ; refresh                    │
│4 │             600     ; retry                      │
│5 │             86400   ; expire                     │
│6 │             3600    ; default TTL                │
│7 │@       NS serv01.companyF.local.                 │
│8 │@       NS serv02.companyF.local.                 │
│9 │serv01  A  192.168.99.1                           │
│10│serv02  A  192.168.99.2                           │
│11│pc01    A  102.168.99.20                          │
│12│)                                                 │
└──┴──────────────────────────────────────────────────┘


До какого максиально возможного времени зона companyF.local будет работоспособна:
a) До 15:35:13 15.05.2010
b) До 15:20:13 16.05.2010
```
    c) До 15:29:00 16.05.2010
```
d) Время ничем не ограничено


___ Пояснение
В данном случае expire равен 86400 секунд, что соответствует 24 часам. Это означает, что вторичный сервер serv02 сможет обслуживать запросы до истечения этого времени с момента последней успешной синхронизации, которая произошла 15.05.2010 в 15:14:00.
Следовательно, зона companyF.local будет работоспособна до:
15:29:00 16.05.2010 (через 24 часа и 15 минут после последней синхронизации).

### 4. Какая модель репликации используется для поддержания в целостности содержимого зон DNS при хранении зон в файлах?
```
    a) Модель репликации с одним основным участником
```
b) Модель репликации с двумя участниками
c) Модель репликации с четырьмя участниками
d) Модель репликации со мнгогими равноправными участниками


___ Пояснение:
Для поддержания целостности содержимого зон DNS при хранении зон в файлах используется модель репликации, при которой существует один основной (primary) DNS-сервер, на котором хранится оригинальная копия зоны, и один или несколько вторичных (secondary) серверов, которые получают копии зоны через процессы репликации (трансфер зоны).
Это соответствует модели репликации с одним основным участником, где основной сервер вносит изменения в зону, а вторичные серверы синхронизируются с ним.

### 5. На скольких DNS-серверах может размещаться первичная зона DNS?
```
    a) На одном
```
b) На двух
c) На четырех
d) На многих


___ Пояснение:
Первичная зона DNS (primary zone) может размещаться только на одном сервере, который является основным (primary) для этой зоны. Этот сервер содержит оригинальную и единственную копию зоны, в которой могут вноситься изменения.
