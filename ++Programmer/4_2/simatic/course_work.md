**Простой светофор** без детекторов транспорта и кнопок пешеходов. Тогда разберём:

## 1. **Алгоритм работы**
Светофор работает в **циклическом режиме** с фиксированным временем фаз:
- **Красный** – 30 секунд
- **Жёлтый** – 3 секунды
- **Зелёный** – 25 секунд
- **Жёлтый** – 3 секунды

### **Алгоритм в псевдокоде:**
1. Включить **красный** (30 сек)
2. Включить **жёлтый** (3 сек)
3. Выключить красный, включить **зелёный** (25 сек)
4. Включить **жёлтый** (3 сек)
5. Повторить с шага 1

## 2. **Аппаратные средства**
### **Количество сигналов CPU:**
- **3 цифровых выхода (DO)** – для управления лампами **красного, жёлтого, зелёного**
- **0 цифровых входов (DI)** – входные сигналы не требуются

### **Выбор CPU**
Самый простой контроллер, который подойдёт, – **CPU 212** или **CPU 214**, так как они имеют:
- CPU 212: **6 DO** (достаточно для 3 ламп)
- CPU 214: **10 DO** (с запасом, если понадобится расширение)

**Вывод:**
Берём **CPU 212** – самый простой и компактный. Если понадобится расширение, можно взять **CPU 214**.

### **Логика работы светофора в LAD (Ladder Logic) для Simatic S7-200**

#### **Переменные:**
- **Q0.0** – Красный свет
- **Q0.1** – Жёлтый свет
- **Q0.2** – Зелёный свет
- **T37** – Таймер красного (30 сек)
- **T38** – Таймер жёлтого (3 сек)
- **T39** – Таймер зелёного (25 сек)
- **T40** – Таймер жёлтого перед красным (3 сек)

---

### **LAD-логика:**

#### **1. Красный свет (30 сек)**
- **Условие:** Если таймер T40 (жёлтый перед красным) истёк, включаем красный и запускаем таймер T37.
- **Выключаем жёлтый** (Q0.1).

```ladder
|----[ T40 ]----[    ]----( T37 )----|
|                   |                 |
|                   |----( Q0.0 )----|  // Красный свет включён
```

---

#### **2. Жёлтый после красного (3 сек)**
- **Условие:** Когда T37 истекает, включаем жёлтый на 3 секунды (T38).
- **Выключаем красный.**

```ladder
|----[ T37 ]----[    ]----( T38 )----|
|                   |                 |
|                   |----( Q0.1 )----|  // Жёлтый свет включён
```

---

#### **3. Зелёный (25 сек)**
- **Условие:** Когда T38 истекает, включаем зелёный (Q0.2) и запускаем таймер T39.
- **Выключаем жёлтый.**

```ladder
|----[ T38 ]----[    ]----( T39 )----|
|                   |                 |
|                   |----( Q0.2 )----|  // Зелёный свет включён
```

---

#### **4. Жёлтый перед красным (3 сек)**
- **Условие:** Когда T39 истекает, включаем жёлтый (Q0.1) на 3 секунды (T40).
- **Выключаем зелёный.**

```ladder
|----[ T39 ]----[    ]----( T40 )----|
|                   |                 |
|                   |----( Q0.1 )----|  // Жёлтый свет включён
```

---

#### **5. Повторение цикла**
- Когда T40 истекает, снова запускаем T37, включая красный.

---

### **Вывод**
Этот LAD-код организует **циклический режим работы светофора** без датчиков, кнопок и адаптивных алгоритмов.

### **LAD-логика с кнопками пешеходов**

#### **1. Обработка кнопок пешеходов**
- Если нажата кнопка пешехода (I0.0 или I0.1), устанавливаем флаг запроса пешеходов (M0.0).

```ladder
|----[ I0.0 ]----[    ]----( M0.0 )----|
|                   |                   |
|----[ I0.1 ]----[    ]----( M0.0 )----|
```

#### **2. Основной цикл светофора**
- Основной цикл остаётся таким же, но с добавлением проверки флага запроса пешеходов (M0.0).

#### **3. Переход к пешеходному зелёному свету**
- Если флаг запроса пешеходов (M0.0) установлен, после завершения текущей фазы включаем пешеходный зелёный свет (Q0.3) и запускаем таймер T41.

```ladder
|----[ M0.0 ]----[    ]----( T41 )----|
|                     |               |
|                     |----( Q0.3 )---|  // Пешеходный зелёный свет включён
```

#### **4. Возврат к основному циклу**
- После завершения таймера T41, сбрасываем флаг запроса пешеходов (M0.0) и возвращаемся к основному циклу.

```ladder
|----[ T41 ]----[    ]----( M0.0 )----|
```
